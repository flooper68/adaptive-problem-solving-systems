# Problem grooming

Use this work session when production, inspection, or assembly evidence may
change a current problem or when the manufacturing lead needs to decide which
problem warrants the next response.

1. Frame the affected goal, problem files, trigger, participants, authority, and
   useful stopping point.
2. Read the system strategy and each problem's evidence, desired change,
   strategy, signal, and grooming history together with current task files,
   relevant sessions, manufacturing knowledge, and material native inspection or
   assembly evidence.
3. Groom one problem at a time. Ask the next load-bearing question about goal
   relevance, evidence, impact, framing, signal, or comparative urgency.
4. Propose `retain`, `revise`, `address now`, or `close`. The manufacturing lead
   approves work selection, reframing, accepted risk, and closure.
5. Record the evidence considered, approved result, rationale, source reference,
   unresolved question, and next trigger in the affected problem file's grooming
   history. Update affected task state and link both from the session record;
   leave raw evidence in its native inspection, machine, assembly, or discussion source.
6. Ensure selected tasks list the active problem IDs they address. On closure,
   set `status: closed`, record the reason and evidence, and move the problem
   file under `problems/archive/`.

Do not create a separate session-summary file unless the native source is not
recoverable or one session produces shared evidence that cannot be represented
clearly in the affected problem files.
