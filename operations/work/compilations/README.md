# Framework compilation candidates

Each compilation run creates a dated directory here containing a sealed
candidate framework artifact and a report of its evidence boundary, provenance,
inference, uncertainty, validation state, and differences from the currently
published framework.

Candidates are `operations-work` evidence. They are not normative APSS content
and must not overwrite `framework/` until the framework-adaptation process has
validated the candidate and the maintainer has approved publication.

Use `<date>-<short-slug>/` for a run. Retain failed or rejected candidates when
they provide material evidence; record their disposition instead of silently
deleting them.
