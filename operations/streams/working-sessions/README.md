# Working sessions

This ongoing stream preserves one file per material brainstorming,
problem-grooming, or other declared work-session invocation. A record identifies
the session, participants, affected problems and tasks, material decisions or
evidence, and stopping point. Problem and task files remain authoritative for
current state; session records preserve what happened.

Use dated descriptive filenames and frontmatter rather than maintaining an
exhaustive index. Do not create a record for trivial activity that changes no
state and contributes no material evidence.

The stream begins with the historical
[initial framework kickoff](2026-07-09-initial-framework-kickoff.md). Its
[resulting definition](2026-07-09-initial-framework-definition.md) is retained
as a separate hidden expected-output fixture: it belongs to the session's
provenance, but must remain outside the source boundary of a clean-room
compilation replay. These historical fixtures remain valid session records.
