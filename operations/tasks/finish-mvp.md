# Define, prune, finish, and exercise the APSS MVP

## Intended result

Establish the smallest coherent MVP, remove or defer nonessential surface,
finish the retained responsibilities, and run one real need through the full
loop into a changed later operation.

## Approach

Refine the system from top to bottom:

1. system goal and strategy;
2. active problems, their strategies, and problem grooming;
3. tasks and task processes;
4. execution, validation, learning, and adaptation; and
5. supporting artifacts only when the retained loop requires them.

Use the initial APSS on Framework Operations, simplify it until the maintainer
can understand and run it, and add complexity later only in response to
feedback.

## Stop condition

The MVP boundary and deferrals are explicit, retained artifacts validate, and
the maintainer can understand and run the complete loop. Evidence from that run
shows whether a subsequent operation improved.

## Current state

The strategy, problem, and task layers have been simplified. `STRATEGY.md`
lives beside `SYSTEM.md`; `PLAN.md` and the generic work log are removed;
candidate tasks live in `tasks/backlog/`, and material history lives in
working-session records.

The [open problems, strategy, and MVP simplification
session](../streams/working-sessions/2026-07-10-open-problems-strategy-and-mvp-simplification.md)
is retained at its accepted stopping point.

Next: continue the top-down review at execution, validation, learning, and
adaptation, classifying each remaining concept as MVP, deferred, or removed.
